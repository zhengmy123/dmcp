## 概述

本文档描述了 Dynamic MCP Server 的所有 REST API 接口。

### 基础信息

- **Base URL**: `http://localhost:8080`
- **数据格式**: JSON
- **字符编码**: UTF-8

---

## 认证方式

### JWT 认证

大部分管理接口需要使用 JWT Token 进行认证。

**请求头格式**:
```
Authorization: Bearer {jwt_token}
```

### vauth_key 认证

MCP 协议接口使用 vauth_key 进行认证。

---

## 统一响应格式

所有接口的响应都遵循以下格式：

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 状态码说明

| code | 说明 |
|------|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 500 | 服务器内部错误 |

## 附录

### 工具名称验证规则

工具名称必须符合以下规则：
- 只能包含小写字母 (a-z)
- 可以包含数字 (0-9)
- 可以包含下划线 (_)
- 长度在 3-64 个字符之间

### 用户角色

| 角色 | 说明 |
|------|------|
| admin | 管理员，拥有所有权限 |
| user | 普通用户，权限受限 |

### MCP Server 类型

| 类型 | 说明 |
|------|------|
| local | 本地服务器 |
| http | HTTP 远程服务器 |
